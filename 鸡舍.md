# CycleFitting -- 基于 MediaPipe 的实时人体尺寸测量系统

## 设计文档（机设）

---

## 一、项目概述

### 1.1 项目背景

传统人体尺寸测量依赖手工量具（软尺、测距仪等），存在效率低、误差大、需要专业人员操作等问题。在服装定制、运动科学、康复医学等领域，快速准确获取人体多维度尺寸具有重要应用价值。

随着深度学习与计算机视觉技术的成熟，基于单目/双目摄像头的非接触式人体测量成为可能。本项目 **CycleFitting** 利用 MediaPipe Holistic 模型进行实时人体姿态检测，结合深度相机（Intel RealSense D455）或普通摄像头，实现 **20+ 项身体尺寸的自动化实时测量**。

### 1.2 项目目标

| 目标 | 描述 |
|------|------|
| 实时性 | 30 FPS 实时姿态检测与测量 |
| 准确性 | 身高测量误差 ≤ 2cm（深度相机模式） |
| 易用性 | 即插即用，支持普通摄像头零配置运行 |
| 完整性 | 覆盖身高、肩宽、臂展、腿长、坐高等 20+ 项测量 |
| 双模态 | 深度相机 3D 测量 + 普通摄像头像素比例测量 |

### 1.3 技术选型依据

| 组件 | 选型 | 理由 |
|------|------|------|
| 姿态估计 | MediaPipe Holistic | Google 开源，33 个身体关键点 + 21×2 手部关键点，无需训练，CPU 实时推理 |
| 深度相机 | Intel RealSense D455 | 搭载 RGB + 深度 + IMU，USB 3.0 即插即用，SDK 成熟 |
| Web 框架 | Flask + Socket.IO | 轻量级，WebSocket 实时推送帧数据，前后端分离 |
| 信号处理 | NumPy + SciPy | 科学计算标准库，滤波/统计函数完备 |
| 配置管理 | YAML + 环境变量 | 分层配置，支持运行时覆盖 |

## 三、核心算法设计

### 3.1 姿态检测流程

```
摄像头帧 (BGR 640×480)
        │
        ▼
┌───────────────────┐
│  BGR → RGB 转换    │
└───────────────────┘
        │
        ▼
┌───────────────────┐
│ MediaPipe Holistic │
│ model_complexity=1 │
│ detection_conf=0.7 │
│ tracking_conf=0.5  │
└───────────────────┘
        │
        ├──→ PoseResult: 33 个身体关键点 (x, y, z, visibility)
        ├──→ LeftHandResult: 21 个左手关键点
        └──→ RightHandResult: 21 个右手关键点
        │
        ▼
┌───────────────────┐
│ 手部-身体关键点融合 │
│ 手腕替换 body[15/16]│
│ 手指替换 body[17-22]│
└───────────────────┘
```

### 3.2 双模态测量方案

本系统支持两种测量路径，根据可用硬件自动选择：

#### 模式 A：深度相机 3D 测量（RealSense D455）

```
RGB 帧 + Depth 帧
        │
        ├──→ MediaPipe → 2D 关键点
        │
        ├──→ 深度图预处理
        │    ├── 空洞填充 (cv2.inpaint)
        │    ├── 双边滤波 (保边去噪)
        │    └── 5×5 中值滤波
        │
        ├──→ 人体分割 (BodySegmentation)
        │    └── 生成掩码，排除背景深度
        │
        └──→ 坐标变换 (CoordinateTransformer)
             ├── 2D 像素 + 深度值 → 3D 世界坐标
             ├── 深度一致性校验（躯干参考深度）
             ├── 异常深度修正
             └── 时序平滑
                    │
                    ▼
            3D 关键点 → Skeleton3D → LinearMeasurements
                    │
                    ▼
            20+ 项身体尺寸 (cm)
```

#### 模式 B：像素比例测量（普通摄像头）

```
RGB 帧
        │
        ├──→ MediaPipe → 2D 关键点 (像素坐标)
        │
        └──→ 像素比例法 (_calculate_from_pixel_landmarks)
             ├── 参考物：鼻尖 → 肩部中心（平均 24.0cm）
             ├── 垂直尺寸：real_cm = (px / ref_px) × 24.0
             ├── 水平尺寸：引入针孔相机模型
             │    ├── 距离估算：d = (ref_real × focal) / ref_px
             │    └── real_cm = (px × d) / focal
             └── 可选：用户校准（已知身高反推参考距离）
                    │
                    ▼
            20+ 项身体尺寸 (cm)
```

### 3.3 关键点稳定化（One-Euro 滤波）

MediaPipe 输出的关键点存在帧间抖动，直接使用会导致测量数据不稳定。系统采用 **One-Euro 自适应低通滤波器**：

- **静止时**：低截止频率 → 强滤波 → 消除抖动
- **运动时**：高截止频率 → 弱滤波 → 低延迟跟踪

```
输入关键点 (x, y, z)
        │
        ▼
┌───────────────────┐
│ 计算变化速度 dx    │
│ dx = (x - x_prev) │
│      / dt          │
└───────────────────┘
        │
        ▼
┌─────────────────────────────┐
│ 自适应截止频率               │
│ fc = min_fc + speed × beta  │
│ 速度低 → fc 小 → 滤波强     │
│ 速度高 → fc 大 → 滤波弱     │
└─────────────────────────────┘
        │
        ▼
┌───────────────────┐
│ 低通滤波输出       │
│ y = α × x +       │
│     (1-α) × y_prev│
└───────────────────┘
        │
        ▼
稳定的关键点坐标
```

### 3.4 生物力学约束验证

测量结果经过人体比例约束验证，确保物理合理性：

| 约束类型 | 规则 | 来源 |
|----------|------|------|
| 肢体比例 | 上臂:前臂:手 ≈ 1:0.86:0.57 | 人体测量学 |
| 左右对称 | 左右对应段长度差 ≤ 10% | 解剖学 |
| 身体比例 | 坐高/身高 ≈ 0.52 | 人体工学 |
| 关节角度 | 膝关节伸展角度 170°-180° | 运动学 |

不满足约束的测量值会被标记低置信度，但仍保留输出供参考。

### 3.5 智能数据采集状态机

```
┌──────────┐
│   IDLE   │  ← 等待启动
└────┬─────┘
     │ 启动采集

┌──────────────┐
```

### 4.2 身高计算算法

身高计算采用 **Y 轴逐段累加法**，针对深度相机和普通摄像头分别处理：

```
身高 = 头部补偿 + Σ(各段垂直投影) + 脚部补偿

头部补偿: nose → head_top（估计值）
脊柱段:   spine_base → spine_mid → spine_shoulder → head
腿部段:   spine_base → hip → knee → ankle → foot
脚部补偿: ankle → foot_bottom（估计值）
```

对于普通摄像头模式，使用 **像素比例法**：

```
real_height = (height_px / reference_px) × REFERENCE_REAL_CM

其中:
  height_px     = 关键点垂直跨度（像素）
  reference_px  = 鼻尖到肩部中心的像素距离
  REFERENCE_REAL_CM = 24.0 cm（鼻尖到肩部中心的统计平均值）
```

## 五、信号处理管线

### 5.1 完整处理链

```
原始输入
    │
    ▼
[深度图预处理]──→ 空洞填充 → 双边滤波 → 中值滤波
    │
    ▼
[人体分割]──→ 姿态关键点 → 轮廓多边形 → 深度掩码
    │
    ▼
[2D→3D 坐标变换]──→ 深度采样 → 异常修正 → 时序平滑
    │
    ▼
[关键点稳定化]──→ One-Euro 自适应滤波
    │
    ▼
[手部融合]──→ 手部关键点替换身体手腕/手指
    │
    ▼
[骨骼模型构建]──→ Skeleton3D（关节点、骨骼段、关节角度）
    │
    ▼
[线性测量计算]──→ 20+ 项尺寸
    │
    ▼
[生物力学约束验证]──→ 置信度评分
    │
    ▼
[多帧融合]──→ 去极值 → 加权平均 → 最终结果
```

### 5.2 深度图预处理

| 步骤 | 算法 | 参数 | 目的 |
|------|------|------|------|
| 空洞填充 | `cv2.inpaint` (NS 方法) | 半径 5px | 填补深度图缺失区域 |
| 双边滤波 | `cv2.bilateralFilter` | d=9, σColor=75, σSpace=75 | 保边去噪 |
| 中值滤波 | 5×5 窗口 | - | 消除椒盐噪声 |
| 人体掩码 | 轮廓多边形 + 深度阈值 | 距离 ±30cm | 排除背景深度干扰 |

## 十一、创新点总结

| 创新点 | 说明 |
|--------|------|
| **双模态测量** | 深度相机 3D 测量 + 普通摄像头像素比例测量，零硬件门槛 |
| **多层信号处理** | 深度预处理 → 人体分割 → 坐标变换 → One-Euro 滤波 → 生物力学约束，五层信号处理管线 |
| **自适应滤波** | One-Euro 滤波器根据运动速度自适应调节，静止时强滤波消除抖动，运动时弱滤波低延迟跟踪 |
| **智能采集** | 稳定性检测 + 倒计时 + 多帧融合（去极值取平均），确保单次测量可靠性 |
| **手部融合** | 手部检测器与身体检测器的关键点融合，实现手指级精度测量 |
| **像素比例法** | 基于人体解剖学比例的无深度相机测量方案，利用鼻尖-肩距作为参考物 |
| **生物力学约束** | 基于人体测量学数据的比例验证，自动识别异常测量并标记置信度 |
| **双端应用** | Web 端（Flask+SocketIO）+ 桌面端（OpenCV），适配不同部署场景 |
